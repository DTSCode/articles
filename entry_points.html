Entry Points


Despite the essential differences of all programming languages, they share at least one thing in common: an entry point. An entry point is the point in the source code where execution starts. Historically speaking, the entry point is a function (method, procedure, routine, etc) with the name (or some variation of) main. In certain cases (with languages like brainfuck or bash) the entry point is not a function at all (and as such, has no name). Instead execution starts at the very beginning of the source file. In cases where the entry point is a function, command line arguments are typically recieved there.
